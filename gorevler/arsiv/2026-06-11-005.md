Feature Export Mimarisi spec'i — Adım 5/7. Önce 2026-06-11-004 tamamlanmış olmalı.

features/layout/ altında:
- PanelShell: sidebar + header + outlet kabuğu (mevcut panel layout yapısından çekilir).
- ErrorPage: 404 / 403 / 500 hata sayfaları tek bileşen, variant prop'u ile.
- Metinler useTranslation('sc-layout') üzerinden, ScLayoutKeys ile uyumlu.
- Playground'daki not-found, error, yetkisiz, hata-ornegi sayfaları ErrorPage'i tüketir.

Doğrulama: playground görsel kırılma yok, lint + tsc temiz.

## Spec

### Açık Sorular
- **i18n altyapısı (`useTranslation`/`ScLayoutKeys`/`sc-layout` namespace) bu repoda henüz yok.** `grep` ile `useTranslation`, `ScLayoutKeys`, `sc-layout` bulunamadı; mevcut `AuthShell`/`SystemMessage` metinleri hardcoded Türkçe. Görev tanımı bu altyapının **2026-06-11-004 (Adım 4/7)** tarafından kurulduğunu varsayıyor. Bu spec, 004'ün şunları teslim ettiğini kabul eder: (a) `useTranslation(ns)` hook'u, (b) `ScLayoutKeys` tip birliği, (c) `sc-layout` namespace çeviri kaynağı, (d) playground'da bir i18n provider. **004 bu sözleşmeyi farklı sağladıysa, bu spec'teki anahtar adları/hook imzası güncellenmeli.**
- **`next/navigation` bağımlılığı:** Mevcut `PanelShell` `useRouter().push()` kullanıyor; ama UI library feature'ı (bkz. `AuthShell` "Next'e bağlı değil") framework-agnostik olmalı. Bu spec, internal `useRouter`'ı kaldırıp `onNavigate?: (path: string) => void` prop'una çevirmeyi öneriyor. Onaylanmazsa `next` peerDependency olarak bırakılır.
- **`hata-ornegi` sayfası:** Bu bir hata sayfası değil, error boundary'yi tetikleyen demo. "ErrorPage'i tüketir" ifadesini, `error.tsx`'in (ki ErrorPage kullanır) tetiklenmesi olarak yorumluyorum; `hata-ornegi/page.tsx` tetikleyici olarak kalır. Aksi isteniyorsa belirtilmeli.

### Bağlam
- Feature Export Mimarisi'nin 5/7. adımı. Feature'lar `@servicecoreui/ui/custom` subpath'inden export ediliyor (`packages/ui/src/custom.ts`), `features/auth/AuthShell` precedent'i mevcut.
- `features/layout/` klasörü var ama **boş**.
- Taşınacak kaynak: `apps/playground/app/_components/PanelShell.tsx` (+ `.module.css`) — şu an playground-local, içinde gömülü mock veri (menü, kullanıcı, bildirim, timer, arama) var.
- Hata sayfaları halihazırda `SystemMessage` (custom) üzerinden çalışıyor: `not-found.tsx` (404), `error.tsx` (500); `yetkisiz/page.tsx` (403) ise farklı olarak `AuthShell` kullanıyor. ErrorPage bu üçünü tek bileşende birleştirecek.

### Hedef Mimari
**`packages/ui/src/features/layout/PanelShell/`** — `PanelShell.tsx`, `PanelShell.module.css`, `index.ts`
- Mevcut shell'in chrome yapısı (top bar + sidebar + body + CommandPalette). Sadece `@servicecoreui/ui` + `/wraps` + `/custom` bileşenleri ve Carbon ikonları kullanılır; doğrudan `antd` import edilmez (mevcut kurala sadık).
- **Veri prop-driven olur, mock'lar default olarak dışa verilir** ki playground görselce birebir kalsın:
  - Props: `children`, `activeNav?`, `navItems?`, `createMenu?`, `userMenu?` (`{name,email,initials,items,actions}`), `notifications?`, `timers?`, `search?` (`{recent,filters,placeholder}`), `onNavigate?`, `lang?`/`onLangChange?`.
  - Mevcut gömülü diziler `defaultPanelNav`, `defaultUserMenu` vb. olarak export edilir; prop verilmezse bunlar kullanılır.
- **Chrome metinleri** (tooltip/aria: "Ara", "Takvim", "Ayarlar", "Yardım", "Bildirimler", "Menüyü genişlet/daralt", "Yeni oluştur", dil etiketleri) `useTranslation('sc-layout')` üzerinden, `ScLayoutKeys` ile tip-güvenli alınır. Domain nav etiketleri (Görev/Çağrı/Olay…) çeviri kapsamı dışı, veri olarak kalır.
- `next/navigation` kaldırılır; `Settings` butonu vb. `onNavigate?.("/ayarlar")` çağırır.

**`packages/ui/src/features/layout/ErrorPage/`** — `ErrorPage.tsx`, `index.ts` (gerekirse `.module.css`)
- `SystemMessage`'ı temel alan tek bileşen, `variant: "404" | "403" | "500"` prop'u.
- Variant'a göre default `code`/`title`/`description` ve default aksiyon(lar) `useTranslation('sc-layout')`/`ScLayoutKeys` üzerinden gelir:
  - `404`: kod 404, "Ana sayfaya dön" aksiyonu.
  - `403`: kod 403, "Girişe dön" aksiyonu (içerik `yetkisiz` sayfasından; tam-ekran `SystemMessage` görünümü).
  - `500`: kod 500, "Tekrar dene" + "Ana sayfa".
- Override prop'ları: `title?`, `description?`, `action?` (tamamen özelleştirme), ve 500 için `onRetry?: () => void`. Link bileşeni framework-agnostik kalsın diye aksiyonlar default'ta `onNavigate?`/`onRetry?` callback'leri üzerinden; playground `next/link` veya `router` ile sarar.

**Export:** `packages/ui/src/custom.ts`'e eklenir:
```
export * from "./features/layout/PanelShell";
export * from "./features/layout/ErrorPage";
```

**Playground tüketimi:**
- `apps/playground/app/layout.tsx` / `DocsShell` ve panel demo sayfaları PanelShell'i `@servicecoreui/ui/custom`'dan import eder; `onNavigate={router.push}` geçirir; eski `_components/PanelShell.tsx` silinir (referansları güncellenir).
- `not-found.tsx` → `<ErrorPage variant="404" onNavigate={…} />`, `error.tsx` → `<ErrorPage variant="500" onRetry={reset} onNavigate={…} />`, `yetkisiz/page.tsx` → `<ErrorPage variant="403" onNavigate={…} />`, `hata-ornegi` tetikleyici olarak kalır (error.tsx üzerinden dolaylı tüketim).
