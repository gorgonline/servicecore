# SolveAI — Kaynaklı arka-plan değerlendirici (Reflexion "Self-Reflection / Msr" parçası)

## Amaç
Çözülen ticket'ları arka planda, KAYNAĞI görerek değerlendiren ve sonucu didim'e otomatik "sözel ders" olarak veren bir değerlendirici bağla. Bugün didim sadece insan teknisyenin 👎 yorumundan öğreniyor; otomatik değerlendirici yok (Reflexion'ın Self-Reflection/Msr parçası eksik).

## Bağlam / gerekçe
- Araştırma yasası Y22 (Reflexion, arXiv:2303.11366): ajanı ağırlık güncellemeden sözel yansımayla öğret — dene → DEĞERLENDİR → sözel ders → hafıza → enjekte. Üç rol: Actor (üretir) + Evaluator (skorlar) + Self-Reflection (ders çıkarır).
- Bizdeki eşleşme: solve = Actor, kalite = Evaluator, didim = Self-Reflection.
- A270 denetimi P1-d: kalite yargıcı "cevap kaynağa dayalı mı" diyor ama KAYNAKLARI görmüyor → uydurma/faithfulness ölçemiyor.
- Reflexion Tablo 3 uyarısı: değerlendirici dayanaksız/zayıfsa, üstüne kurulan yansıma BOZAR (0.52 < 0.60 base). Önce değerlendiriciyi dayanaklı yap, sonra didim'e bağla.
- Mevcut durum (koddan doğrulandı): kalite solve'a BAĞLI DEĞİL (karar/node.py:20 — runtime hard-gate olarak iyi önerileri reddettiği için bilinçli çıkarılmış). Reddedilen rol "kapı"ydı; "arka-plan değerlendirici" rolü hiç denenmedi. Bu görev o ikinci rolü kuruyor.

### Faz 1 — değerlendiriciyi dayanaklı + arka-plan yap
**A) kalite paketi (görevdeki 4 nokta):** `kaynaklar_metin` parametresini imza → payload → parse_node state → verdict_ver inject zincirinden geçir. inject'te mevcut desenle aynı satır: `payload.append(f"Kaynaklar:\n{kaynaklar}")`. **`""` default zorunlu** (geriye uyum: kaynaksız çağıran kırılmasın — `canli.py` mevcut çağrı, integration testleri).

**B) caller-side wiring (5. nokta — Açık Soru 1):** Kaynak metnini OneriGirdi'den (`benzerler[].cozum` + `cozum_adaylari[].icerik`) **düz string'e** çeviren saf formatter yaz. **A2A sınırı:** kalite, solve şemasını import edemez (departmanlar yalnız A2A/string konuşur) → formatter solve/panel tarafında, kalite sadece string alır. OneriGirdi'yi solve run'ından yüzeye çıkar (öneri: `coz_ve_izle` `oneri_hazirla` event çıktısını yakalasın → `kaynaklar_metin` döndürsün), çağıranda `uyum_denetle(ITSM_UYUM_KURALLARI, oneri_metni, kaynaklar_metin)` ile geç.

**C) non-blocking:** Asıl cevap zaten kalite'den bağımsız (kalite solve grafiğinde yok). Faz 1 için kaynak görünürlüğü esas; panel `/coz` awaitini arka-plana almak (fire-and-forget) Açık Soru 2'ye bağlı UX kararı — persistana karar verilince netleşir.

- **Faz 2 dikkat (kod okumadan görülmeyen riskler):**
  - `itsm_geri_bildirim_metni` her sonuc için **SON** sinyali alır ve yalnız 'olumsuz' ise dahil eder (`feedback_kaynak.py:24-34`). Otomatik 'olumsuz' yazımı insan 👍/👎 kanalını ezebilir/eziLEbilir → **ayrı kaynak** seçeneği daha güvenli.
