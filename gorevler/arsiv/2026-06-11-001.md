## Spec

### Açık Sorular
1. **`features/layout/` içeriği:** Görev `features/layout/` klasörünü açmayı istiyor ama bu adımda içine taşınacak bileşen belirtmiyor. Mevcut layout-vari bileşenler (`UserMenu`, `NotificationCenter`, `PageHeader`, `RecentPanels`, `NavCard`, `SearchableMenu`) "ve diğer tüm mevcut bileşenler → `components/`" kuralına giriyor. **Varsayım:** Bu adımda `features/layout/` yalnızca boş iskelet (`.gitkeep` + ileride doldurulacak `index.ts` placeholder) olarak açılır; layout bileşenleri ileriki adımlarda (2/7…7/7) taşınır. Onaylanmazsa adım listesi güncellenir.
2. **Auth-komşusu bileşenler:** `PasswordChecklist`, `SystemMessage`, `SettingsForm` auth/ayarlar akışıyla ilişkili ama görev SADECE `AuthShell`'i `features/auth/` altına taşıyor. **Varsayım:** Bunlar `components/` altında kalır (görev metnine sadık). 

### Bağlam (araştırma bulguları)
- `panel/packages/ui/src/` altında ~78 bileşen klasörü + `theme/` (kalır) + 6 kök dosya var: `index.ts`, `wraps.ts`, `custom.ts`, `charts.ts`, `icons.ts`, `css-modules.d.ts`.
- **Barrel dosyaları** (`index.ts`, `wraps.ts`, `custom.ts`, `charts.ts`) bileşenlere `./Button`, `./Charts/...` gibi göreli yolla referans veriyor → bunlar `./components/...` olacak.
- **Bileşenler arası importlar `../X` biçiminde** (ör. `DataTable` → `../Button`, `../Table`). Tüm bileşenler aynı `components/` klasörüne taşındığı için bu kardeş importlar **olduğu gibi geçerli kalır** — kütlesel taşımada içe import düzenlemesi GEREKMEZ.
- **Tek istisna `AuthShell`:** `features/auth/AuthShell/` altına inince `../Heading`, `../Text`, `../Brand` importları kırılır → `../../../components/{Heading,Text,Brand}` olarak güncellenmeli. Ayrıca `custom.ts` içindeki `export * from "./AuthShell"` → `./features/auth/AuthShell` olacak.
- **`theme/`** ve **`icons.ts`** kalır; bileşenler theme'i göreli importla çekmiyor (CSS değişkenleri + `@carbon/icons-react` paket importu kullanıyor) → etkilenmez.
- **`build.mjs` entryPoints** kök dosyaları (`src/index.ts`, `src/icons.ts`, `src/theme/index.ts`, `src/wraps.ts`, `src/custom.ts`, `src/charts.ts`) işaret ediyor; bunlar src kökünde kaldığı için **build config değişmez**.
- **Public API / package.json `exports` değişmez** — `components/` ve `features/` tamamen iç organizasyon; tüketici (playground) hâlâ `@servicecoreui/ui`, `/wraps`, `/custom`, `/charts`, `/icons`, `/theme` subpath'lerini kullanır. **Davranış değişikliği yok.**

### Strateji
Taşımalar `git mv` ile yapılır (geçmiş korunur). Sıra: önce kütlesel taşıma + barrel düzeltme (kardeş importlar bozulmaz), sonra tek istisna olan `AuthShell`'in feature taşıması + import düzeltmesi, en son boş `features/layout/` iskeleti. Her aşamadan sonra `typecheck`/`build` ile doğrulanır.
